### **Основные функции AxenAPI:**

1. **Визуализация графа системы**:
   - Пользователь может видеть элементы системы (сервисы, топики, события) в виде графа.
   - Граф показывает связи между компонентами (например, кто производит и читает сообщения из топиков).

2. **Управление компонентами системы**:
   - **Сервисы**: Создание, переименование и удаление сервисов.
   - **Топики**: Создание, редактирование (включая выбор типа брокера), удаление и фильтрация топиков.
   - **События**: Создание, редактирование и управление событиями.

3. **Работа с брокерами сообщений**:
   - Управление списком брокеров (добавление, удаление, редактирование).
   - Поддержка различных типов брокеров (например, Apache Kafka, RabbitMQ, JMS, Other).

4. **Фильтрация и поиск**:
   - Пользователь может фильтровать объекты по тегам для быстрого поиска нужных компонентов.

5. **Информационная панель**:
   - Детальная информация о выбранном компоненте (например, сервисе или топике) доступна через информационную панель.

6. **Редактирование JSON Schema**:
   - Возможность создания, редактирования и экспорта JSON Schema для событий.
   - Генерация примеров данных на основе JSON Schema.

7. **Экспорт и импорт спецификаций**:
   - Пользователь может импортировать существующие спецификации системы и экспортировать текущую конфигурацию в форматах (например, DOC, PDF).

8. **Детализация топиков**:
   - Пользователь может раскрыть детали топика, чтобы увидеть все события, которые он производит или читает.

9. **Управление ссылками**:
   - Возможность добавлять, редактировать и удалять ссылки на брокеры сообщений.

10. **Печатные формы**:
    - Выгрузка спецификаций в удобочитаемом формате (например, DOC или PDF) для предоставления заказчику.

---

### **Что такое AxenAPI?**
AxenAPI — это **инструмент для проектирования, документирования и управления архитектурой распределенных систем**, таких как микросервисы или системы обмена сообщениями. Он предоставляет пользователю удобный интерфейс для работы с графическим представлением системы, управления её компонентами и связями, а также для создания технической документации.

---

### **Ключевые особенности AxenAPI:**
- **Гибкость**: Поддержка различных типов брокеров и возможность настройки JSON Schema.
- **Наглядность**: Визуализация системы в виде графа и информационной панели.
- **Удобство**: Простые способы редактирования компонентов (переименование через двойной клик, контекстное меню и т.д.).
- **Документирование**: Возможность выгрузки спецификаций в форматах DOC и PDF.

---

### **Целевая аудитория:**
AxenAPI ориентирован на:
- Архитекторов систем, которые проектируют взаимодействие между сервисами.
- Разработчиков, которые реализуют микросервисы и системы обмена сообщениями.
- Технических писателей или аналитиков, которым нужно создавать документацию для заказчиков.
